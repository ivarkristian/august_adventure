# GitHub Setup Checklist

Use this after creating the public repository.

## Repository

1. Create public repo (suggested name: `august-adventure`).
2. Push this project to `main`.

## Labels

Create labels:

- `august-feedback`
- `bug`
- `feature`
- `playtest`
- `triage-needed`
- `accepted`
- `wontfix`

## Branch Protection (main)

- Require pull request before merging.
- Require status checks to pass.
- Include `CI / test` check.

## Milestones (optional)

- `M1 - Core Loop`
- `M2 - Puzzle Depth`
- `M3 - Content Expansion`

## OpenCode project initialization

Run `/init` from inside this repository after first push. Then review `AGENTS.md` and keep the role rules above intact.
